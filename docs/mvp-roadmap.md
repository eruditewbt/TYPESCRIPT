# 30-Day MVP Roadmap

The first shipping target is a focused MVP for the TypeScript SaaS Starter Platform.

## Goal

Ship a working starter platform that proves three things:

- students can learn through real contributions
- the product is useful enough to sell
- the system can grow into the larger academy ecosystem

## Week 1: Foundation and Repo Setup

- Finalize monorepo structure
- Define product boundaries for CLI, API, and web app
- Establish shared package contracts
- Decide on backend and frontend framework defaults
- Write contributor onboarding docs and issue templates

## Week 2: Core Platform Slice

- Implement API skeleton
- Implement web app skeleton
- Implement CLI command structure
- Add authentication foundations
- Add base domain models: user, organization, project

## Week 3: Vertical Feature Delivery

- Connect web app to API
- Add role-based access basics
- Add starter billing abstraction
- Add at least three real dashboard workflows
- Deploy a preview environment

## Week 4: Learning and Beta Launch

- Record onboarding lessons for the first contribution path
- Create guided student tasks mapped to real issues
- Publish landing page and starter-kit offer
- Run a beta cohort using the live repo
- Collect feedback and prioritize the next sprint

## MVP Definition of Done

- Monorepo is organized and documented
- CLI can scaffold or bootstrap at least one template
- API exposes working auth and organization endpoints
- Web app supports login and basic project navigation
- Students can take an issue from docs to pull request
- There is at least one paid offer attached to the product
