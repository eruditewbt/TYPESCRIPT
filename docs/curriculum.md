# Curriculum Framework

The curriculum should be delivered as a progression from learner to contributor to creator.

## Level 1: Apprentice

- Focus: syntax, types, functions, modules, async flows, Git basics
- Work type: bug fixes, typed utilities, starter tasks
- Outcome: can contribute safely to guided issues

## Level 2: Builder

- Focus: APIs, React, state, validation, testing, deployment basics
- Work type: feature implementation with review support
- Outcome: can build production features within the platform

## Level 3: Collaborator

- Focus: package design, architecture, code review, shared systems
- Work type: cross-package contributions and team delivery
- Outcome: can work as part of a real TypeScript product team

## Level 4: Creator

- Focus: productization, monetization, growth, operations
- Work type: launch templates, plugins, or SaaS extensions
- Outcome: can turn skills into revenue-generating assets

## Lesson format

Every module should follow the same pattern:

1. Learn the concept.
2. Inspect the live codebase.
3. Complete a scoped task.
4. Submit the work for review.
5. Deploy or publish the result.
6. Reflect on product and business impact.
